## What It Does

1. **Step 1** — User enters name + email
2. **Step 2** — Upload credit report / supporting docs
3. **Step 3** — Review and confirm dispute items
4. **Step 4** — Pay $25 (Cash App / Venmo / Zelle)
5. **Step 5** — Confirmation number + 30-day countdown timer + next steps

## Deploy to Render (Copy-Paste-Enter)

### Option A — Upload files directly to GitHub (no terminal needed)

1. Go to your GitHub repo: `github.com/Businessbear1981/BitmojiGuy5MinCreditFix`
2. Delete all existing files (they're placeholders)
3. Upload ALL files from this zip
4. Go to Render → Manual Deploy → Clear build cache → Deploy

### Option B — From PowerShell (inside the repo folder)

```powershell
# Make sure you're IN the repo folder first:
cd C:\Users\sgill\BitmojiGuy5MinCreditFix

# Copy all files from this zip into that folder, then:
git add .
git commit -m "Full app build - AE Labs AE.CC.001"
git push
```

Render will auto-detect `render.yaml` and deploy.

## Environment Variables (set in Render dashboard)

| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY` | Yes | Flask session key (auto-generated by render.yaml) |
| `SMTP_HOST` | Optional | For email confirmations (e.g. smtp.gmail.com) |
| `SMTP_PORT` | Optional | Default 587 |
| `SMTP_USER` | Optional | Your email address |
| `SMTP_PASS` | Optional | App password |
| `FROM_EMAIL` | Optional | Sender email address |

Without SMTP vars, the app works fine — it just skips sending emails and logs to console.

## API Endpoints

| Method | Path | Purpose |
|---|---|---|
| GET | `/` | Main app |
| POST | `/api/start` | Create session (step 1) |
| POST | `/api/upload` | Upload documents (step 2) |
| POST | `/api/review` | Confirm dispute items (step 3) |
| POST | `/api/pay` | Confirm payment (step 4) |
| GET | `/api/status/:confirmation` | Check status by confirmation # |
| GET | `/health` | Health check |

## Integrate Real Payments (Future)

Replace the `/api/pay` route stub with:
- **Stripe**: `stripe.PaymentIntent.create(amount=2500, currency='usd')`
- **Square**: Square Payments API
- The route already accepts `payment_confirmed: true` — wire it to your webhook
